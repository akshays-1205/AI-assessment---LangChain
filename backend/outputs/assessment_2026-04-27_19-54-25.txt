Q2: How would you approach optimizing the accuracy versus latency tradeoff for a deep learning model using quantization strategies such as FP16 and INT8, and what metrics would you use to evaluate the effectiveness of these optimizations?
A2: I’d start with a FP32 baseline, then benchmark FP16 and INT8 variants using ONNX Runtime/TensorRT to measure real latency and accuracy deltas.
FP16 is typically the first step since it gives speedups with minimal accuracy loss, while INT8 is applied with careful calibration or QAT if sensitivity is high.
I’d use mixed-precision fallback (keeping sensitive layers like first/last or normalization in higher precision) to recover accuracy where INT8 struggles.
Effectiveness is evaluated using latency (p50/p95), throughput, memory footprint, and task metrics (top-1/F1), selecting the best Pareto point under SLA constraints.

Q3: Can you elaborate on how you determine which specific layers, such as the first or last layers, or normalization layers, should be kept in higher precision using mixed-precision fallback, and what role do you think calibration or QAT plays in making this decision for INT8 quantization?
A3: I determine sensitive layers using a mix of activation statistics + ablation testing: I compare FP32 vs INT8 output drift per layer and look for high error accumulation, often in first conv, last classifier, and normalization blocks.
First/last layers are usually kept in FP16/FP32 because they directly affect input representation and final decision boundaries, so small quantization noise gets amplified.
Normalization layers (BatchNorm/LayerNorm) are sensitive because their small variance values can explode under INT8 scaling, so they’re often excluded or fused carefully.
Calibration helps estimate stable activation ranges, while QAT actively adapts weights to quantization noise—together they decide whether INT8 is sufficient or fallback is required.
